# 保姆级第三方平台接入配置指南 (Feishu & TAPD)

这份文档是一份**手把手、保姆级**的实操手册。不讲复杂的底层代码，只讲**如何去第三方平台（飞书、TAPD）注册应用、如何获取配置凭证、如何打通 Webhook**，让这个机器人能真正在你的公司里跑起来。

---

## 🟢 第一部分：飞书机器人 (Feishu Bot) 接入与配置

我们要让机器人能在飞书群里收发消息，并在企业内部生效。

### 1. 注册并创建企业自建应用
- **前往后台**：打开 [飞书开发者后台 (Feishu Open Platform)](https://open.feishu.cn/app/)。
- **登录并创建**：点击“创建企业自建应用”，随便起个好听的名字（比如 `AI 交付助理`）并上传个头像。
- **获取凭证**：
  - 进入应用详情页，点击左侧导航栏的 **「凭证与基础信息」**。
  - 找到 **`App ID`** 和 **`App Secret`**。
  - **👉 填入本项目的** `config.json` -> `feishu.app_id` 和 `feishu.app_secret` 中。

### 2. 开通机器人能力与权限
- 点击左侧 **「添加应用能力」** -> 选择 **「机器人」**，点击添加。
- 点击左侧 **「权限管理」**，至少搜索并开通以下权限（否则机器人收不到消息、也发不出看版）：
  - **消息与群组**：`获取与发送单聊、群组消息` (`im:message.receive_v1`, `im:message.send_as_bot`)
  - **云文档空间**：`查看、评论、编辑和管理云空间所有文件` (`drive:drive`)、`上传文件` (`drive:file:upload`)
  - **权限设置**：`更新文档权限` (`docs:permission.setting:write_only`)

### 3. 如何配置长连接 (事件订阅)
**什么是长连接？** 简单来说，我们的机器人程序在启动时，会主动去找飞书的服务器建立一根“专线”。这根线是双向互通的，有人发消息飞书就会顺着这根线推下来。因此，**你不需要配置公网 IP，也不需要搞内网穿透**。

- **开启长连接**：
  - 回到飞书开发者后台，点击左侧 **「事件订阅」**。
  - 在顶部的 **订阅方式** 中，选择 **「使用长连接接收事件」**（推荐）。
  - 点击“保存”。
- **订阅具体事件**：
  在“事件订阅”页面下方，点击“添加事件”，搜索并勾选：**「接收消息 v2.0」**（事件类型为 `im.message.receive_v1`）。

### 4. 发布应用
- 点击左侧 **「版本管理与发布」** -> 创建一个新版本 -> 点击“申请发布”。（如果你是企业管理员，可以直接审核通过）。发布后，这台机器人才算真正活过来。

### 5. 如何获取群组的 Chat ID (`group_chat_id`)
最简单的笨办法：
1. 在飞书里建一个群，把你的机器人拉进群里。
2. 在群里随便 `@机器人 嗨`。
3. 查看你本地服务终端打印出的日志（或者前端的日志面板），里面会打印出这条消息的原始 JSON，其中 `chat_id: "oc_xxxxxxxxxx"` 就是这个群的 ID。复制它填到你的群组配置里即可。

## 🟠 第二部分：TAPD 系统打通与配置

TAPD 是腾讯出品的敏捷研发平台，我们需要让机器人有权限直接进去建单和改状态。

### 1. 申请 TAPD API 凭证
- **前往后台**：打开 [TAPD 开放平台](https://www.tapd.cn/company/feature_store/list?type=api) 或者找你公司的 TAPD 管理员。
- **获取 API 密钥**：
  - 在“公司管理” -> “开放平台 API” 中，新建一个 API 账号。
  - 你会获得 **`API 账号 (API Token / Username)`** 和 **`API 密钥 (API Password)`**。
  - **👉 填入本项目的** `config.json` -> `tapd.api_token` 中（如果系统要求拼接，一般是填 API Token）。
  *(注：如果你的公司采用的是 API V1 / OAuth 的方式，也可以生成固定的 Access Token。)*

### 2. 获取 Workspace ID (项目 ID)
这是最简单的！
- 打开你的 TAPD，进入你需要机器人建单的项目页面。
- 观察浏览器的网址，例如：`https://www.tapd.cn/52052188/bug/`
- 其中 **`52052188`** 就是这个项目的 **`workspace_id`**。
- **👉 填入** `config.json` -> `tapd.workspace_id`。

### 3. 获取工作项类型 (Workitem Type ID)
如果你的项目用的是自定义需求 / 自定义任务，你需要告诉系统建哪种类型的单子：
- 你可以通过调用 TAPD 的接口或者抓包查看你要建的“Story”或“Task”在后台的数字 ID，比如 `1152052188001000017`。
- **👉 填入** `config.json` -> `tapd.workitem_type_id`。

## 🔵 第三部分：开源大模型 (LLM) 配置

机器人能够听懂人话，全靠外接的大语言模型（目前代码中默认通过 OpenAI 兼容接口调用 DeepSeek ）。

- **前往平台**：例如去 [DeepSeek 开放平台](https://platform.deepseek.com/) 注册账号。
- **创建 API Key**：在后台生成一个 `API Key`（通常是 `sk-xxxxxx` 开头的长串）。
- **配置信息**：
  - `api_base`: 填入模型提供商的 URL，例如 `https://api.deepseek.com/v1`。
  - `api_key`: 填入刚才获取的 sk-xxxxx。
  - `model`: 填入具体的模型名，比如 `deepseek-chat`。

> **开源 SDK 引用说明**：
> 项目在底层接入大模型时，使用的是 Vercel 提供的重量级开源包 `@ai-sdk/openai` ([官方地址](https://sdk.vercel.ai/providers/ai-sdk-providers/openai))。通过这个标准化的库，我们可以一键无缝替换国内外任何兼容 OpenAI 格式的大模型。

## 🎯 总结验收

完成上面这三大块，你的系统就真正具备了“眼、手、脑”：
1. 飞书长连接（WebSocket）给了它 **“眼睛和嘴巴”**（看群聊、发消息）。
2. TAPD API 给了它 **“手”**（写工单、改状态）。
3. DeepSeek / LLM API 给了它 **“大脑”**（分析你说的话是建单还是闲聊）。

所有的秘钥和配置找齐后，只需在项目自带的 **React 可视化前端管理台** 里把它们一填，点击保存。你在群里 @ 机器人，它就能立刻开始帮你干活了！
